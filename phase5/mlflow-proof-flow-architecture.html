MLflow Proof Flow (Evidence -> Gate -> Promote/Reject)

How training/eval evidence is produced, stored in MLflow, and consumed by CI/CD decisions

What is “proof”?

  • • Metrics logged by eval code
  • • Params (model/config/data version)
  • • Artifacts (reports, predictions, model files)
  • • Run metadata (commit SHA, timestamps)

Why MLflow matters

  • • Central ledger for model evidence
  • • Reproducibility and audit trail
  • • Comparable runs before promotion
  • • CI gates consume objective data

Operator rule

  • • No logged evidence = no promotion
  • • Always link deploy to run_id
  • • Failed gates trigger retrain loop
  • • Policies decide, not intuition
